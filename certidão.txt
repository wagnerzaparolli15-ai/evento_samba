1. Estrutura de Comunicação (O Fluxo)
O sistema segue uma linha reta para evitar quebras:

Venda: index.html ➔ pagamento.html (Gera Pix).

Confirmação: templates-feedback.html (Aguarda aprovação) ➔ obrigado.html (Gera o Ingresso).

Entrada: portaria.html (Equipe vê quem pagou) ➔ recepcao.html (Libera o cliente).

Consumo: bar.html (Cliente faz pedido) ➔ admin_total.html (Você vê o lucro).

2. Variáveis de Ambiente (As "Chaves" no Render)
Para o projeto nunca mais "esquecer" quem ele é, estas chaves devem estar sempre no painel Environment do Render:

DATABASE_URL: O caminho do seu banco de dados Postgres.

MP_ACCESS_TOKEN: O token do seu Mercado Pago.

SECRET_KEY: A senha interna que mantém as sessões de login seguras.

3. Manutenção Rápida (Caso precise agir)

Esqueci a senha do Admin? No código app.py, o login padrão é sempre criado na inicialização como wagner / 123.

Erro 502/500 apareceu? Vá ao Render, clique em Manual Deploy ➔ Clear Build Cache & Deploy. Isso limpa a "sujeira" e reinicia o motor.

O QR Code não sai? Verifique se o MP_ACCESS_TOKEN no Render não tem espaços vazios antes ou depois do código.

4. Comando de Inicialização (Start Command)
No Render, em Settings, o comando deve ser sempre:
gunicorn app:app
